《javascript dom 编程艺术》笔记

发表于 2017-11-09

1~3章

  1. getElementsByClassName()
    • 这个方法知道但是不怎么用。这一小节里收获最大的是当老浏览器不支持新方法时，如何优雅地回退。
  2. P51如何阻止链接被点击的默认行为
    • 事件处理函数机制：在某个元素添加了事件处理函数后，一旦事件发生，相应的JavaScript代码会被执行。被调用的JavaScript会返回一个值，这个值会传递给事件处理函数。
    • onclick函数根据返回值判断这个链接是否被点击。感觉这个点还不是特别懂，感觉和浏览器渲染是不是有点关系。顺带查了一下a标签的onclick事件：
      1. <a href="javascript:js_method()">在这种写法里，点击会执行js_method()的相应代码，但是这会把js方法暴露在地址栏中。
      2. <a href="javascript:void(0)">void(0)返回undefined，地址不发生跳转。
      3. <a href="javascript:;">执行了一条空的js代码

第四章

  1. 如果一个站点用到多个js文件，为了减少对站点的请求次数，应该把这些js整合到一个文件里。
  2. childNodes属性用来获取任何一个元素的所有子元素，返回一个包含所有子元素的数组。返回的是所有类型节点，不仅仅是元素节点。
  3. window.onload在页面加载时运行。
  4. nodeType获取节点属性。
    • nodeType=1表示该节点是元素节点
    • nodeType=2表示该节点是属性节点
    • nodeType=3表示该节点是文本节点。
  5. nodeValue获取节点的值。

第五章

这一章主要讲的是为了合理使用js应该尽量遵守的原则，看书的时候才意识到自己以前写代码是有多随意、多粗糙。

  1. 平稳退化。要考虑到不支持或禁用了js功能的浏览器访问网页时的效果。
  2. 结构与样式的分离。
  3. 渐进增强
  4. 对象检测。在使用dom方法的时候要先检验方法是否存在。

第六章

  1. addLoadEvent(func)作者提出的addLoadEvent()很实用

第七章

  1. docunment.write、innerHTML
    • 以前添加元素的时候，就是用相对简单粗放的innerHTML。诚然，深感如作者所言，innerHTML无法获取HTML文档中的细节，也不够灵活。
  2. 在HTML中添加元素，其实就是修改ＤＯＭ树，涉及到的方法有createElement、appendChild、createTextNode。
  3. 把元素插入到想要的位置，可以使用ＤＯＭ提供的insertBefore,和自己编制的insertAfter。
  4. Ajax异步加载页面内容的技术
    • Ajax的优势是把页面的请求以异步方式发给服务器，服务器在后台处理请求，在这个过程中用户还能继续与页面交互。但是使用Ajax，爬虫无法抓取相关内容，用户也无法通过浏览器的后退按钮回到之前的页面。
      1. XMLHttpRequest对象。js通过这个对象发送请求，同时也处理响应。 2. getHTTPObeject获取XMLHttpRequest对象，创建出来的对象可以通过open方法指定服务器上将要访问的文件。
      2. onreadystatechange会在服务器给XMLHttpRequest对象送回响应的时候被触发执行。其中，发回响应时，XMLHttpRequest对象的readyState会被更新。这个属性有5个值，当readyState=4时，就可以访问服务器发送回来的数据了。这个数据就是前面open方法打开的文件里面的。
      3. 发回来的数据存放在XMLHttpRequest的两个属性中：responseText（保存文本字符串属性的信息）和responseXML（是一个DocumentFragment对象，还不太能理解这是啥）

第八章

这一章主要讲的是获取同一类元素的某一个属性，通过js构建列表，将它们显示出来。

第九章

  1. 通过改变元素的className来改变元素的样式。尽量不要直接使用.style.保证结构-样式-行为尽可能都是分离的。
  2. 作者编制的addClass函数
  3. 通过减少函数内硬编制的常数（包括元素的类型、样式、数值）让函数更抽象，适用范围更广。

第十章

用js函数实现动画效果，个人感觉没有css3的animation来得要简洁方便。但是比较灵活，比较随心所欲吧。

第十一章

  1. 着重介绍了HTML5的一些新的标记元素，canvas，video。介绍了一下怎么用js改变这些新标记元素的默认样式，当其中有些属性不支持时，如何回退。
  2. Modernizr一个js库，可以检测浏览器是否支持某项功能，可以根据它在css中定义相应的增强或退化版本

第十二章

这一章没有新的知识点，就是以搭建一个网站为例走一遍网页开发的过程。确定站点结构-确定页面结构-用html完成结构-用css完成样式-用js增强交互效果。作者叙述的过程就是按照渐进增强的思路来走的，每一步都是对上一步的完善。相比于平时自己写网页时，总是一个元素的html、css、js完成了再去写下一个元素，这样的方式要有条理的多，也不太会出现拆东墙补西墙的问题。日后要慢慢转变习惯。

附录

js库。看书上的源码时可以明显感觉到库用起来有多方便，代码量一下少了一大半。看了一下介绍，对jQuery和Prototype比较感兴趣。消化消化，可以准备开《锋利的jQuery》了。
