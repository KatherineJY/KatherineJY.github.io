hexo again

发表于 2017-11-08

最近电脑坏掉了，所以很多东西都要重新配置，包括重新搭一下hexo。
所以对于一个已经在git上的项目，该怎么使用hexo呢？

我以前一直错误地以为hexo -d会把整个博客的文件夹传上去，但其实并不是的。对比了一下文件夹和git上的东西，发现并没有package.json、node_modules、_config.yml以及包含原始的md文件的source文件夹。
所以，其实push到远程仓库上的只是生成后的页面文件。所以直接把远程仓库上的博客文件夹下下来，会发现里面并没有包含有hexo的配置文件，即package.json。

  • 在git bash中通过cnpm先安装hexo-cli（用npm真的是慢到伤不起，一度以为是自己断网了）
  • 在博客文件夹中建立一份package.json（从github上下下来的文件夹里是没有的），随便拷一份就好了
  • cnpm install hexo -> cnpm install -> cnpm install hexo-deployer-git

  • 正常使用hexo
