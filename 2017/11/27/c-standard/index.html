关于C编程的一些规范

发表于 2017-11-27

摘自《现代编译原理》

程序设计风格

在用C表示树数据结构时,遵循以下一些约定：

  1. 树都用文法表示
  2. 一棵树用一至多个typedef来表示,每个typedef对应文法中的符号
  3. 每个typedef定义一个指向相应struct的指针，这个struct的名字以下划线结束，它除了在typedef的声明中和该结构定义本身中出现外，绝不会在其他地方使用。（TODO:不是很明白为什么要有这条要求 这个指针具体在哪里会用到呢？？）
  4. 每一个struct有一个kind成员和u成员.kind是一个指明不同选择的枚举量，每个枚举值对应一个可选的文法规则。
  5. 如果有一个规则的右部有多个非平凡的符号，则它的union有一个本身也是结构的成员给出组成它的这些值。(TODO:不是很明白为什么会多个非平凡值的情况。一个union配一个kind？？)
